SQL

Niveau : Confirmé

Ma définition

Le langage SQL (Structured Query Language) a été créé en 1974 par Donald D. Chamberlin et Raymond F. Boyce. Il est utilisé pour la communication avec des bases de données relationnelles. Il permet notamment de lire, créer, modifier et supprimer des données (CRUD) et peut par exemple servir à stocker des utilisateurs avec leurs informations.

Mes éléments de preuve

L’apprentissage du SQL est une obligation, Chaque projet informatique utilisant une base de données nécessite l’utilisation du langage SQL.

C’est pour cela que j’ai commencé le SQL dès la première année.

J’ai pu le pratiquer sur chaque projet auquel j’ai participé que ce soit en entreprise ou en école.

Pour pratiquer le SQL il vous faudra sûrement un logiciel permettant l'administration et le requêtage de base de données. Mon favori est DBeaver, il possède ce que tous les autres ont de mieux avec des raccourcis clavier très pratique.
Il en existe d’autres comme : MySQL, PostgreSQL et SQL Server Management Studio (SSMS).

Le projet MACS GDPR est un logiciel métier qui stocke et manipule une grande quantité de données. Par exemple, lors du dépôt d’une demande de voirie, nous stockons l’équivalent du Cerfa-14023*01, avec l’utilisateur à l’origine de la demande et l’heure du dépôt.

Pour conserver ces données nous avons dû concevoir une base de données, si nous reprenons l’exemple plus haut, nous avons besoin de deux tables une pour les permissions de voirie et une autre pour les utilisateurs. La table des permissions de voirie contient une colonne avec l’identifiant de l’utilisateur qui l’a créée. On appelle ça une clé étrangère, ces deux tables ont une relation 1 -> N, ou OnetoMany.

Un autre exemple issu d’un projet mandaté par INCITE Bordeaux et Libourne, où nous souhaitons stocker l’état des lieux d’un logement. Pour élaborer un peu plus mes propos, voici un diagramme représentant la structure des tables nécessaires aux fonctionnalités.

Une table « Bâtiment » contient l’ensemble des logements, la table « logement » contient le détail d’un logement. La table « logement » contient également l’identifiant du bâtiment auquel le logement appartient.

Bref rappel, chaque colonne a un type, soit du texte, un nombre ou un booléen.

Les champs surlignés en jaune sont les clés primaires, ce sont les identifiants uniques de chaque table des logements ou bâtiments.
Les champs surlignés en rouge sont les clés étrangères, ce sont les identifiants uniques d’une autre table.

Grâce à mes 5 années d’expérience, mes compétences en SQL ne se limitent pas aux SELECT, INSERT, UPDATE, DELETE, je suis capable de créer des fonctions, des sélections complexes avec des conditions, des fonctions « triggers » et des vues.

Mon autocritique

Je le répète mais savoir utiliser le SQL est une obligation, il s’agit du langage de requête le plus utilisé dans le monde, il est présent dans la majorité des projets informatiques.

J’estime pouvoir intégrer et aider une équipe dans la conception d’une base de données ou dans l’évolution d’une base existante. Mais également participer au développement de requêtes pour des nouvelles fonctionnalités.

Le SQL est un langage efficace avec une syntaxe lisible, il est donc facilement compréhensible et suffisamment documenté pour les débutants.
Aujourd’hui je me sens à l’aise avec ce langage, cela renforce ma confiance au sein d’une équipe de développeurs. Uniquement la pratique permet d’acquérir une véritable compréhension du langage.
