Git

Niveau : confirmé

Ma définition

Git est une technologie gratuite et open source qui permet une gestion de versions des fichiers d’un projet. Il permet un travail collaboratif plus efficace et il est utilisé à grande échelle dans le développement informatique.

C’est pour ces raisons qu’il est devenu la référence pour la gestion de version dans les projets. Aujourd’hui, je considère qu’il n’est pas normal de ne pas utiliser Git pour un projet de développement informatique, surtout si plusieurs développeurs interviennent dans le projet.

Mes éléments de preuve

Personnellement, je n’ai jamais participé à un projet de développement sans utiliser Git. J’ai eu l’occasion de découvrir Git dès le premier semestre au sein du Groupe AEN, nous l’avons utilisé pour le projet Keep Clean.

Une fois en alternance j’ai pu approfondir mes compétences avec Git étant donné que l’entreprise possède son propre serveur Git avec tous ses projets. J’ai créé mes propres projets avec une version de développement et une version de production. Le fait de dupliquer le projet en une version de développement et de production implique la création de « merge request » (fusion des fichiers) entre les deux projets. J’ai également appris à résoudre des conflits entre fichiers, à ignorer l’envoi de fichier, la création de « branche » et de « fork ».

Par exemple, on me demande de développer une nouvelle fonctionnalité ou bien de corriger un bug, je récupère le projet je corrige le code puis je peux envoyer mes modifications. Il m’est aussi arrivé plusieurs fois d’utiliser l’historique de changement d’un fichier pour mieux comprendre l’origine d’un nouveau bug.

Mon autocritique

Git rend un réel service à l’équipe de développement, si je dois commencer un nouveau projet le premier réflexe sera de créer un répertoire Git. Aujourd’hui je considère être confirmé, avoir tous les bons réflexes avec Git, je maîtrise l’ensemble des fonctionnalités disponibles. Pour tous ceux qui souhaitent apprendre à utiliser Git, je vous conseille fortement d’utiliser le jeu Oh My Git!, c’est assez ludique et il permet de comprendre toutes les manipulations possibles avec Git.
