Epop, le hip hop 2.0

Contexte

Dream Landes Dance est une association de danse HIPHOP située dans le sud-ouest, elle est menée par un professeur de danse Mathieu Peyroutet.

Mathieu Peyroutet souhaite améliorer ses méthodes de travail pour lui et ses élèves il a donc pensé à une solution pour que chaque élève puisse avoir une application sur son mobile utilisable pendant les cours de danse. Avant le déploiement du site web responsive, les élèves devaient noter sur un cahier leurs mouvements et chorégraphies. Le problème de saisir les mouvements et chorégraphies sur papier est qu’ils pouvaient perdre leurs notes, et, ne pouvaient pas facilement les partager avec les autres élèves. Idem pour le professeur qui ne pouvait pas partager facilement les chorégraphies.

L’objectif du projet était donc de réaliser une application web responsive afin de moderniser les façons de stocker et gérer des mouvements de danse. Pouvoir créer des groupes d’utilisateurs, des « troupes de danse » pour être plus précis, diffuser les actualités de l’association facilement à tous les utilisateurs et avoir un panel d’administration pour gérer les utilisateurs, les rôles, les paramètres disponibles dans les listes à choix et gérer les actualités affichées.

Etant donné qu’il y avait le « le festival des cultures urbaines » du 17 au 19 mai 2019 à Dax, Mathieu souhaitait que l’application soit prête pour la présenter aux festivaliers.

Le plus gros risque était simplement la capacité de réalisation du projet, nous étions seulement 4 étudiants au semestre 2, dans lequel nous commencions à apprendre à utiliser PHP et l’utilisation des bases de données.

Le développement des projets scolaires du Groupe AEN – INTECH sont gratuits pour le commanditaire, cependant au moment du déploiement il y a un risque que le commanditaire fasse marche arrière quand il faut payer pour réserver le nom de domaine et pour l’hébergement du projet.

Travaux réalisés

Avant de commencer la réalisation du projet nous devions bien comprendre les méthodes de travail du professeur de danse et ses élèves, pour proposer une solution capable de répondre à leurs besoins. Pour cela nous avons été invités à un cours du soir de Hip-hop. Après avoir saisi les besoins, nous avons pu passer à la conception du projet suivie de la planification et répartition des tâches avec un outil comme Tom’s Planner.

Lors de la conception nous avons dû réfléchir à toutes les fonctionnalités qui seront mises à disposition au sein de l’application, avec les différents types de comptes (admin, professeur, élèves).
Nous avons dû réfléchir à une identité pour l’application, avec une charte graphique, définir un logo et des visuels.

Avant de débuter le développement concret du projet, nous avons d’abord réalisé des diagrammes d’architectures, qui représentent un modèle conceptuel de données. Cela permet de créer plus facilement, avec l’aide d’un schéma, les tables et leurs relations dans la base de données.

Les technologies du projet sont PHP pour la partie Backend et JS HTML CSS pour la partie Frontend.
Nous avons d’abord créé l’environnement de travail ensemble, une fois le projet déposé sur Git nous avons commencé à nous attribuer les tâches. Personnellement j’ai commencé par préparer l’authentification des utilisateurs via un formulaire de connexion, puis je me suis chargé de la partie « Chorégraphie ».

Les chorégraphies sont une liste de mouvements à enchainer les uns après les autres, la fonctionnalité devait donc permettre de créer/modifier/supprimer une chorégraphie et d’ajouter/supprimer des mouvements déjà présents dans la liste des mouvements de l’utilisateur, mais aussi de pouvoir déplacer la position des mouvements dans la chorégraphie.

Pour lister les chorégraphies, les mouvements, les utilisateurs présents en base de données, nous avons utilisé Datatable qui est une librairie JavaScipt, cette librairie permet de facilement gérer la pagination et la recherche.

La chorégraphie est définie par un nom, un statut (public, privée) et par un battement par minute. Le statut permet de partager facilement la chorégraphie avec sa troupe. Nous avons aussi ajouté une fonction pour « lancer » la chorégraphie :

Cela permet au danseur de poser son téléphone au sol et de pouvoir pratiquer la chorégraphie en ayant les bons rythmes.

Nous avons aussi intégré l’envoi de mail avec PHP Mailer, pour confirmer un compte lors de l’inscription mais aussi pour notifier les utilisateurs pour un rendez-vous de troupe, une modification sur une chorégraphie ou l’ajout d’actualité.

Une fois le site web responsive prêt nous avons accompagné Mathieu Peyroutet pour l’achat du nom de domaine et l’hébergement sur OVH, nous avons aussi laissé une documentation technique destinée aux futurs développeurs, une documentation d’utilisation pour les utilisateurs et une documentation pour OVH destinée au commanditaire (Mathieu), pour qu’il puisse gérer de manière autonome son environnement OVH.

Pour l’instant le site est toujours en ligne si vous souhaitez y jeter un coup d’œil : epop.

Cependant la majorité des fonctionnalités ne sont accessibles, sauf si vous êtes connecté à un compte, mais ce sera à l’administrateur de valider votre compte. Etant donné que le site est destiné aux adhérents de l’association Dream Landes Danse il y a fortes chances que vous ne soyez pas accepté.

Résultat

J’ai trouvé le projet très attractif dès le départ, l’idée de développer un outil destiné à des passionnés de danse est une source de motivation pour fournir une solution adéquate. Lors des dernières réunions avec Mathieu Peyroutet, il nous a confié que ce projet était « son petit bébé », nous sommes fiers d’avoir pu lui fournir une solution qui correspond à ce qu’il souhaitait après 5 mois de travail.

Personnellement j’ai pu approfondir mes connaissances en web responsive, découvrir PHP et le SQL pour utiliser une base de données MySQL.

Nous avons aussi pu participer au festival des cultures urbaines, où nous avons pris le micro pour faire une petite présentation de l’application.

Nous avons pu faire plaisir aux jeunes passionnés de Hip-hop, qui préfèreront sans doute utiliser une application mobile plutôt qu’un carnet pour partager et noter leurs chorégraphies et mouvements.
